## 3️⃣ ผลลัพธ์สุดท้าย (Final Output)

1. **ฐานข้อมูลบทความ** – `Docs/literature_database.csv`  
   - 284 รายการ, คำสั่งค้นหา, เมตาดาต้าเต็ม

2. **รายการบทความที่คัดเลือก** – `Docs/selected_papers.md`  
   - 48 งานที่ตรงกับเกณฑ์ (ปี 2019‑2024, มีมาตรฐาน CO₂)

3. **Literature Matrix** – `Docs/literature_matrix.md`  
   - สรุป **Research Objectives**, **Methodology** (Scope, Standard), **Key Findings**, **Limitations**, **Standard Referenced** ของแต่ละงาน  

4. **Gap Analysis** – `Workspace/Decision/gap_analysis.md`  
   - **ช่องว่างที่พบ**  
     1️⃣ **Scope 3 ของบรรจุภัณฑ์พลาสติกในประเทศไทย** – ไม่มีงานวิจัยใดที่ประเมินการปล่อยก๊าซจากการขนส่งวัตถุดิบและการกำจัดขยะ (Scope 3) อย่างครบวงจร  
     2️⃣ **การเปรียบเทียบมาตรฐานสากล (ISO 14067 / PAS 2050) กับ Thai Carbon Footprint Label** – งานเปรียบเทียบเชิงปริมาณยังขาดแม้แต่ 1 งาน  
     3️⃣ **บรรจุภัณฑ์โลหะและแก้ว** – มีเพียง 2‑3 งานที่ทำ LCA ทั้ง Scope 1‑3; ส่วนใหญ่จำกัดอยู่ที่ Scope 1‑2  
     4️⃣ **การประเมินระดับผลิตภัณฑ์ (per‑unit) vs ระดับโรงงาน** – ส่วนมากเน้นระดับโรงงาน; ขาดงานที่ให้ค่าการปล่อย CO₂ ต่อตารางเมตรหรือต่อกิโลกรัมบรรจุภัณฑ์

## 4️⃣ ข้อสังเกตและข้อเสนอแนะ (Observations & Recommendations)

| ประเด็น | คำอธิบาย | ข้อเสนอแนะ |
|--------|----------|------------|
| **1. ความครบถ้วนของ Scope 3** | งานวิจัยส่วนใหญ่ให้ความสำคัญกับ Scope 1‑2; มีเพียง 7 % ของบทความที่ครอบคลุม Scope 3 อย่างเต็ม | - ใน Phase 2 (Data Collection) ให้เน้นการเก็บข้อมูล Supply‑Chain (วัตถุดิบ, การขนส่ง, การกำจัด) <br>- พิจารณาใช้ฐานข้อมูล **Ecoinvent** หรือ **GaBi** เพื่อนำมาเป็นตัวอย่างการคำนวณ |
| **2. ความแตกต่างของมาตรฐาน** | มีการอ้างอิงหลายมาตรฐาน (ISO 14067, PAS 2050, Thai Label) แต่ไม่มีการเปรียบเทียบผลลัพธ์โดยตรง | - พัฒนา **Matrix of Standard Comparison** ในขั้นตอนวิเคราะห์ต่อไป <br>- ใช้กรณีศึกษา 2‑3 โรงงานเพื่อประเมินผลต่างระหว่าง ISO 14067 กับ Thai Label |
| **3. ประเภทบรรจุภัณฑ์ที่ยังขาดข้อมูล** | บรรจุภัณฑ์พลาสติก, โลหะ, แก้ว มีบทความน้อยกว่า 10 งานรวมกัน | - สร้าง **Target List** ของโรงงานที่ผลิตบรรจุภัณฑ์โลหะ/แก้ว เพื่อทำการสำรวจภาคสนาม <br>- หากข้อมูลอาจหายาก ให้ทำ **Systematic Review** ของ literature สากล (อังกฤษ/ยุโรป) แล้วทำ **Contextual Mapping** ไปยังประเทศไทย |
| **4. รูปแบบการสรุปข้อมูล** | Literature Matrix ปัจจุบันเป็น Markdown ธรรมดา – ยังไม่สามารถนำเข้าเครื่องมือวิเคราะห์อัตโนมัติได้ | - แปลงเป็น **CSV/Excel** (col: DOI, Year, Scope, Standard, CO₂ per unit, Limitations) เพื่อให้ Analyst Agent ใช้ต่อได้ง่าย <br>- เก็บไฟล์ต้นฉบับใน `Data/processed/literature_matrix.xlsx` |
| **5. การจัดการความเสี่ยงด้านลิขสิทธิ์** | มีการอ้างอิงบทความที่อาจไม่มีการเปิดให้ดาวน์โหลดฟรี | - จัดเก็บเพียงเมตาดาต้า + DOI <br>- หากต้องการเต็มเนื้อหา ให้ขอ **Permission** จากผู้เขียนหรือใช้บริการของสถาบันที่มีการเข้าถึง (เช่น ฐานข้อมูลมหาวิทยาลัย) |
| **6. การสื่อสารกับ Agent ตัวอื่น** | Shared Context ถูกบันทึกแล้ว แต่ยังไม่มีไฟล์ **README** ระบุวิธีการใช้ | - สร้างไฟล์ `Docs/Shared_Context_README.md` อธิบายโครงสร้าง JSON และวิธีดึงข้อมูลโดย Python (`json.load`) เพื่อให้ Writer Agent เริ่มเขียนบทที่ 2‑3 ได้ทันที |
