## 1️⃣ วัตถุประสงค์ (Objective) & กระบวนการคิด (Thinking Process)

| วัตถุประสงค์ | รายละเอียด |

### กระบวนการคิด (Thinking Process)  

1. **วิเคราะห์ความต้องการ** – ตรวจสอบข้อกำหนด (ภาษา, โครงสร้าง, ความยาว, รูปภาพ, รูปแบบการอ้างอิง) และระบุ **Skill** ที่ต้องใช้ (SK‑RES‑01 … SK‑RES‑07).  
2. **กำหนดคีย์เวิร์ด** ที่สอดคล้องกับหัวข้อหลัก 3 ส่วน.  
3. **ออกแบบ Workflow** เป็น 7 ขั้นตอน (ค้นหา → คัดกรอง → สกัดข้อมูล → สรุป → จัดเก็บ Context → ส่งต่อ → ตรวจรับ).  
4. **ระบุความเสี่ยง** ที่เคยทำให้ผลงานล้มเหลว (ภาษาอังกฤษ, ความยาวไม่ตรง, รูปภาพ/อ้างอิงไม่ครบ) พร้อม **วิธีบรรเทา** (regex ตรวจภาษา, การคำนวนจำนวนคำ, ตรวจสอบไฟล์ภาพ, ใช้เทมเพลต Word).

### คำแนะนำต่อไปสำหรับทีม

1. **รักษามาตรฐาน QA** – ทำการตรวจสอบอัตโนมัติทุกขั้นตอนสำคัญ (language, word‑count, image existence).  
2. **อัปเดต Bibliography อย่างสม่ำเสมอ** – ใช้สคริปต์ `scripts/update_bib.py` เพื่อดึง citation จาก Scopus API ทุกครั้งที่มีการเพิ่ม paper ใหม่.  
3. **บันทึก Log** ของแต่ละรอบการทำงานใน `Logs/research_log_YYYYMMDD.txt` เพื่อช่วย trace การตัดสินใจและเป็นหลักฐานสำหรับ QA/Advisor.  
4. **ทำการ Review ร่วมกับ Writer Agent** – ก่อนส่งไฟล์ Word ขั้นสุดท้าย ให้ทำ “pair‑review” สั้น ๆ เพื่อให้แน่ใจว่าการอ้างอิงและรูปภาพตรงตามที่กำหนด.

**สรุป**  
Research Agent ได้ดำเนินการครบวงจรตั้งแต่การค้นหาวรรณกรรม → คัดกรอง → สกัดข้อมูล → สรุปเป็นบทที่ 2 (≈ 1,000 คำ) → จัดเก็บ Context ที่ใช้ร่วมกับ Agent อื่น → ส่งข้อมูลให้ Writer Agent ผลิตไฟล์ Word พร้อมรูปภาพและอ้างอิงตามข้อกำหนด. ผลลัพธ์สอดคล้องกับคำสั่งของ Orchestrator และได้แก้ไขปัญหาเดิมทั้งหมดอย่างเป็นระบบ.
