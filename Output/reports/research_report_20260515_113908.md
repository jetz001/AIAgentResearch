### กระบวนการคิดของ Research Agent  

1. **กำหนดความต้องการข้อมูล** – วิเคราะห์ข้อกำหนดของ Orchestrator (โครงสร้าง 1.1‑1.5, มาตรฐานอ้างอิง, คีย์เวิร์ด “อ้างอิง”).  
2. **วางแผนค้นหา** – ใช้ **SK‑RES‑01** (Literature Search) เพื่อระบุแหล่งข้อมูล OBG (CFO) และ ISO 14064‑1/‑2.  
3. **คัดกรอง** – ใช้ **SK‑RES‑02** เพื่อตรวจสอบอายุ (ไม่เกิน 5 ปี), ความเกี่ยวข้อง, คุณภาพของเอกสาร.  
4. **สกัดข้อมูลสำคัญ** – ผ่าน **SK‑RES‑03** ดึงข้อมูลเชิงวิธีการคำนวน GHG, Scope 1‑3, ตัวแปรการประเมินคาร์บอน.  
5. **วิเคราะห์ช่องว่าง** – **SK‑RES‑04** ทำ Gap Analysis เพื่อยืนยันว่าไม่มีข้อมูลสำคัญที่หาย.  
6. **จัดทำโครงสร้างและไฟล์ Word** – ใช้สคริปต์ Python‑docx (ส่วนของ **SK‑RES‑08**) เพื่อแปลง Markdown → .docx, ใส่คีย์เวิร์ด “อ้างอิง”.  
7. **บันทึก Context ร่วม** – **SK‑RES‑07** ทำ Shared Context Management ลง `Memory/Shared/Shared_Context.json` เพื่อให้ Agent อื่นเข้าถึงได้ทันที.

## 4️⃣ ข้อสังเกตและข้อเสนอแนะ  

| ประเด็น | ข้อสังเกต | ข้อเสนอแนะ |
|----------|-----------|-------------|
| **การเข้าถึงมาตรฐาน ISO** | ต้องซื้อหรือสมัครสมาชิกเพื่อดาวน์โหลด PDF ฉบับเต็ม; เราใช้เอกสารสรุปจากฐานข้อมูลมหาวิทยาลัยที่ให้สิทธิ์เข้าถึง. | ควรจัดสรรงบประมาณสำหรับการซื้อหรือสมาชิก ISO เพื่อให้ทีมมีแหล่งข้อมูลฉบับเต็มในโครงการต่อไป. |
| **รูปแบบ citation** | ระบบอัตโนมัติแทรก “อ้างอิง” ได้สำเร็จ, แต่สไตล์อาจแตกต่างจากที่สถาบันกำหนด (APA vs IEEE). | ตรวจสอบคู่มืออ้างอิงของมหาวิทยาลัยก่อนส่งไฟล์ขั้นสุดท้าย; หากต้องการสไตล์เฉพาะให้ปรับสคริปต์ Python‑docx ตาม. |
| **การจัดการ Context** | การบันทึก citation ใน `Shared_Context.json` ทำให้ Agent ต่อไปเข้าถึงได้ง่าย, แต่ไม่มีการ version‑control ของไฟล์นี้. | แนะนำให้ใช้ระบบ Git หรือจัดเก็บเวอร์ชันในไฟล์ (`Shared_Context_v01.json`) เพื่อป้องกันการเขียนทับโดยไม่ตั้งใจ. |
| **ขอบเขตข้อมูลอ้างอิง** | เราใช้แหล่งอ้างอิงหลัก 3 แห่ง; ยังมีบทความวิจัยระดับนานาชาติที่อาจเพิ่มมิติเพิ่มเติม (เช่นการประเมิน LCA ของการพิมพ์). | ในขั้นต่อไป (บทที่ 2‑3) ควรทำ **Systematic Review** เพิ่มเติมเพื่อเสริมความครอบคลุมของวรรณกรรม. |
| **ความสอดคล้องกับมาตรฐานไทย** | โครงสร้าง 1.1‑1.5 สอดคล้องกับแนวทางวิทยานิพนธ์ไทย, แต่บางสไตล์อาจต้องปรับ **หัวข้อ 1.5** ให้เป็น “สรุปบทนำ” ตามที่บางคณะกำหนด. | ตรวจสอบแบบฟอร์มของคณะ/สถาบันก่อนส่งบทที่ 1 ไปยังที่ปรึกษา. |
