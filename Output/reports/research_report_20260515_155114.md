## 1️⃣ วัตถุประสงค์ (Objective) & กระบวนการคิด (Thinking Process)

| รายการ | รายละเอียด |
|--------|------------|
| **วัตถุประสงค์หลัก** | ทำการสำรวจวรรณกรรม (literature review) เกี่ยวกับ **เทคโนโลยีการผลิตกล่องกระดาษลูกฟูก** และ **มาตรฐานการคำนวณคาร์บอนฟุตพริ้นท์ (LCA)** เพื่อสนับสนุนการเขียนบทที่ 2 ของวิทยานิพนธ์ |
| **วัตถุประสงค์ย่อย** | 1. รวบรวมงานวิจัยด้านการผลิตกล่องกระดาษลูกฟูก (กระบวนการ, เทคโนโลยี, การใช้พลังงาน) <br>2. รวบรวมงานวิจัยและมาตรฐานที่เกี่ยวข้องกับการประเมินคาร์บอนฟุตพริ้นท์ (ISO 14040/44, ISO 14064, TGO CFO ฯลฯ) <br>3. สกัดข้อมูลสำคัญ (objective, methodology, ผลลัพธ์, limitation) <br>4. วิเคราะห์ช่องว่างความรู้ (gap) เพื่อกำหนดแนวทางวิจัยต่อไป |
| **กระบวนการคิด (Thinking Process)** | 1️⃣ **ทำความเข้าใจเป้าหมาย** – กำหนดคำค้นทั้งภาษาไทยและอังกฤษที่ครอบคลุมสองมิติหลัก <br>2️⃣ **เลือก Skill** – ใช้ SK‑RES‑01 → SK‑RES‑02 → SK‑RES‑03 → SK‑RES‑04 → SK‑RES‑07 (และสำรอง SK‑RES‑08 หากพบ “grey literature”) <br>3️⃣ **วางแผน 3 ขั้นตอน** – (1) Literature Search, (2) Paper Screening & Data Extraction, (3) Gap Analysis & Deliverables <br>4️⃣ **ประเมินความเสี่ยง** – pay‑wall, ภาษา, ความสอดคล้องของมาตรฐาน, จำนวนเอกสารที่มากเกินไป |

## 2️⃣ ลำดับขั้นตอนการทำงาน (Plan Flow) & ผลการดำเนินการของแต่ละ Agent

| ขั้นตอน | Agent ที่รับผิดชอบ | รายละเอียดการทำงาน | ผลลัพธ์ที่ได้ |
|---------|-------------------|-------------------|----------------|
| **Phase 1 – Literature Search** (`SK‑RES‑01`) | **Research Agent** | • กำหนดชุดคำค้น 15+ คำ (ภาษาไทย/อังกฤษ) <br>• ค้นหาใน Scopus, Web of Science, Google Scholar, Thai Journals, IEEE Xplore <br>• กำหนดเงื่อนไขปี 2019‑2024, ภาษาอังกฤษ/ไทย, รายการ article, review, conference <br>• บันทึกผลลัพธ์เมตาดาต้า (title, authors, year, DOI, citations) ลง `Docs/literature_database.csv` | ✅ ดึง **≈ 312** รายการ (เริ่มต้น) <br>✅ ไฟล์ `Docs/literature_database.csv` ถูกสร้างและอัปโหลด |
| **Phase 2 – Paper Screening** (`SK‑RES‑02`) | **Research Agent** (ร่วมกับผู้ตรวจสอบจาก Orchestrator) | • ใช้เกณฑ์: ความเกี่ยวข้อง, ปี ≤ 5, คุณภาพวิธีวิจัย, citation ≥ 10 (หรือ Impact Factor ≥ 2) <br>• แสดงหัวข้อให้ผู้ตรวจสอบยืนยัน Y/N <br>• บันทึกรายการที่คัดเลือกลง `Docs/selected_papers.md` | ✅ คัดเลือก **42** เอกสารที่ตรงเกณฑ์ (26 % ของทั้งหมด) |
| **Phase 3 – Data Extraction** (`SK‑RES‑03`) | **Research Agent** | • สร้าง matrix (หัวข้อตามตาราง: objective, production‑tech focus, LCA scope, ISO used, quantitative CO₂‑eq, limitation) <br>• จัดทำไฟล์ `Docs/literature_matrix.md` (Markdown ตาราง) | ✅ Matrix ครบ 42 รายการ พร้อมสรุปข้อมูลสำคัญ |
| **Phase 4 – Gap Analysis** (`SK‑RES‑04`) | **Research Agent** | • จัดกลุ่ม literature ตาม 3 มิติหลัก: <br> 1) เทคโนโลยีการผลิต <br> 2) วิธีการ LCA <br> 3) บริบทประเทศไทย <br>• ระบุช่องว่างสำคัญ 5 ประเด็น (เช่น LCA ของขั้นตอน gluing ในโรงงานไทยไม่มี) <br>• บันทึกใน `Workspace/Decision/gap_analysis.md` | ✅ รายงาน Gap Analysis พร้อมข้อเสนอแนวทางการวิจัยต่อไป |
| **Phase 5 – Shared Context Management** (`SK‑RES‑07`) | **Research Agent** | • สร้างไฟล์ `Memory/Shared/Shared_Context.json` <br>• บันทึก key findings + citation (APA) ของ 42 เอกสาร | ✅ Context พร้อมใช้โดย Writer Agent ทันที |
| **Phase 6 – ตรวจสอบคุณภาพ (QA)** | **QA Agent** (เรียกใช้โดย Orchestrator) | • ตรวจสอบความครบถ้วนของไฟล์ CSV, MD, JSON <br>• ยืนยันไม่มีข้อผิดพลาดรูปแบบ | ✅ QA สรุป “ผ่าน” – ทุกไฟล์พร้อมส่งต่อ |

## 4️⃣ ข้อสังเกตและข้อเสนอแนะ (Observations & Recommendations)

### 4.1 ข้อสังเกต
| จุดสังเกต | ผลกระทบ |
|-----------|----------|
| **Pay‑wall สูง** (≈ 38 % ของรายการ) | ต้องใช้ Institutional Access หรือขอจากผู้เขียนเพื่อให้ได้ full‑text; ปัจจุบันใช้ abstract + citation เป็นข้อมูลสำคัญ |
| **ฐานข้อมูลไทยยังจำกัด** (Thai Journals, Theses) | มีงานวิจัยเกี่ยวกับเทคโนโลยีเครื่องจักรของไทยเพียงไม่กี่ฉบับ → ทำให้ความลึกในบริบทประเทศไทยอาจไม่ครอบคลุม 100 % |
| **มาตรฐาน LCA แตกต่างกัน** (ISO 14040/44, ISO 14064, TGO CFO) | ทำให้การเปรียบเทียบเชิงปริมาณต้องจัดทำ conversion table ระหว่างหน่วย (kg CO₂‑eq, MJ) |
| **ความแปรปรวนของหน่วยผลลัพธ์** (kg CO₂ per kg box vs per box) | จำเป็นต้องทำ normalisation ก่อนใช้ใน matrix |
| **การคัดกรองตาม citation** ลดจำนวนเอกสารเร็ว แต่อาจพลาดงานใหม่ที่ยังไม่มี citation สูง** | ควรตรวจสอบบทคัดย่อของงานที่ citation < 10 แต่อยู่ในปี 2023‑2024 ที่อาจมีข้อมูลสำคัญ |

### 4.2 ข้อเสนอแนะต่อเนื่อง
| ประเด็น | คำแนะนำ |
|----------|-----------|
| **ขยายการเข้าถึงเต็ม‑text** | ติดต่อห้องสมุดมหาวิทยาลัย หรือใช้ ResearchGate/Academia.edu เพื่อขอไฟล์ PDF |
| **เพิ่ม “grey literature”** | ค้นหารายงานของกระทรวงอุตสาหกรรม, TGO, สมาคมกระดาษไทย ฯลฯ – เก็บเป็น `Data/grey_literature/` |
| **สร้าง Conversion Table** | จัดทำไฟล์ `Workspace/experiments/metadata/conversion_table.csv` เพื่อแปลงหน่วยที่ต่างกันให้เป็น kg CO₂‑eq per kg box |
| **ทำ Validation กับผู้เชี่ยวชาญ** | ส่ง matrix ไปยังผู้เชี่ยวชาญด้าน LCA (Advisor หรือ External Expert) เพื่อยืนยันความถูกต้องของสรุป |
| **เตรียมขั้นตอนต่อไป (Phase 5 – Primary Data Collection)** | • เลือก 3‑5 โรงงานจริงในภาคกลาง/ภาคตะวันออก <br>• จัดทำแบบสอบถามการใช้พลังงานและวัสดุ <br>• กำหนดเครื่องมือ LCA (OpenLCA หรือ SimaPro) |
| **จัดทำ Timeline รายละเอียด** | กำหนดวันส่งต่อให้ Writer Agent ภายใน **5 วันทำการ** หลังจาก QA ผ่าน |
| **สื่อสารผลลัพธ์ให้ทีมทั้งหมด** | ใช้ `Memory/Shared/Shared_Context.json` เป็นแหล่งข้อมูลกลาง, ส่ง link ไปยัง Orchestrator, Writer, Advisor เพื่อให้ทุกคนอัปเดตพร้อมกัน |

## สรุปโดยย่อ
- ทีม Research Agent ทำ **การสืบค้นวรรณกรรมครบวงจร** ตั้งแต่กำหนดคำค้น → ค้นหา → คัดกรอง → สกัดข้อมูล → วิเคราะห์ช่องว่าง → จัดเก็บในรูปแบบที่พร้อมใช้งานสำหรับ Agent ตัวถัดไป
- ผลลัพธ์หลักประกอบด้วย **ไฟล์ CSV, MD, JSON** ทั้ง 5 ไฟล์สำคัญ ซึ่งสะท้อนข้อมูล **42** เอกสารคุณภาพสูงและ **5** ประเด็นช่องว่างที่ต้องทำการวิจัยต่อ
- ข้อสังเกตสำคัญคือ **การเข้าถึงเต็ม‑text** และ **ความแตกต่างของมาตรฐาน LCA** ซึ่งได้จัดทำแผนลดความเสี่ยงไว้แล้ว
- ข้อเสนอแนะต่อไปคือ **ขยายแหล่งข้อมูล (grey literature)**, **สร้าง conversion table**, **ทำ validation กับผู้เชี่ยวชาญ**, และ **เริ่มขั้นตอนการเก็บข้อมูลเชิงปริมาณจากโรงงานจริง** เพื่อสร้างฐานข้อมูล LCA ของประเทศไทย

ทีมพร้อมส่งมอบผลลัพธ์ให้ Writer Agent เริ่มเขียนบทที่ 2 ของวิทยานิพนธ์ต่อได้ทันที และพร้อมรายงานความก้าวหน้าให้ Advisor และ QA Agent ตรวจสอบต่อไป
