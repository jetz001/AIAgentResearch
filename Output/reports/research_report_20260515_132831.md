## 4️⃣ ข้อสังเกตและข้อเสนอแนะ  

| ประเด็น | คำอธิบาย | ข้อเสนอแนะ |
|----------|-----------|--------------|
| **ความยากในการคัดกรองภาษาอังกฤษ** | ข้อมูลต้นทางบางส่วน (เช่น งานวิจัยสากล) มีเนื้อหาอังกฤษ ซึ่งต้องแปลงเป็นภาษาไทยให้ครบถ้วน | ควรจัดทำ **Glossary** ภาษาไทย‑อังกฤษ สำหรับคำเฉพาะ (เช่น “Scope 1‑3”, “t‑test”) เพื่อให้ Writer Agent แปลได้สอดคล้องและลดความเสี่ยงของการหลุดอังกฤษ |
| **การประมาณจำนวนคำ** | การคำนวณจากสรุปสั้นอาจทำให้จำนวนคำสุดท้ายเกิน/น้อยกว่าวัตถุประสงค์เล็กน้อย | แนะนำให้ใส่ **“word‑budget”** ใน Prompt (เช่น “≈ 500 คำต่อหัวข้อ ± 30 คำ”) และให้ Writer Agent แบ่งส่วนย่อยเพื่อปรับหลังเขียน |
| **การแทรกรูปใน Word** | หาก Writer Agent ใช้ Markdown → Word conversion อาจทำให้ตำแหน่งรูปหลุด | แนะนำให้ Writer Agent ใช้ **python‑docx** หรือเครื่องมือที่สนับสนุนการแทรกรูปโดยตรงจาก path เพื่อรับประกันตำแหน่งที่แน่นอน |
| **การตรวจสอบ QA อัตโนมัติ** | การนับคำและตรวจหาอักขระอังกฤษทำได้โดยสคริปต์ง่าย แต่การประเมิน “ความเชื่อมโยงเนื้อหา” ยังต้องตรวจคน | ควรพิจารณาเพิ่ม **Content Coherence Check** (โดยใช้ LLM ตรวจสอบว่าข้อความทุกหัวข้อมักสอดคล้องกับข้อสรุปที่กำหนด) เพื่อยกระดับคุณภาพก่อนส่งมอบ |
| **การบันทึกประวัติการทำงาน** | ปัจจุบันบันทึกผลสรุปใน `Shared_Context.json` เท่านั้น | แนะนำให้เพิ่ม **Log File** (`Workspace/logs/research_phase.log`) เพื่อบันทึกเวลาที่ทำแต่ละขั้นตอนและผู้รับผิดชอบ ง่ายต่อการ audit ในอนาคต |
